# Finding — core guards are agnostic; reply-content word-regex is a business-layer, multilingual-fragile concern

## Design law (verified in looprun source)
looprun's **core** guards MUST be domain- and language-agnostic. Only **specific** guards (generated by the
`agentspec` skill or hand-written) may carry domain/linguistic content. Verified: `@looprun-ai/core`'s
`guards.ts` holds **zero** word patterns — every linguistic regex (`claimRe`/`askRe`/`offerRe`/`exemptRe`/…)
is **injected** by the domain bundle. The comments say so ("the runtime holds no failure-language of its
own"). The per-verb claim regexes (suspend/resume/transfer/…) live in the **generated lexicon**, not core.

## The limitation to improve
The reply-honesty guards (`noFalseFailureClaim`, `noFabricatedSuccess`, …) detect **semantic** claims ("the
model said it couldn't do X" / "claimed success") via **word-search regex on the reply text**. Even though
that regex is business-owned (allowed), word-search is intrinsically **not multilingual**: an English
`/cannot|can't|unable/` never matches a Portuguese reply → the guard silently dies (false-negative) or
mis-fires. This produced a real false-positive: an honest, policy-required refusal *"I cannot resume — the
contract has ended"* tripped `noFalseFailureClaim` because the lookups that turn had succeeded, even though
`resume_line` was never attempted.

## The correct fix pattern (used in this spec)
Do **not** let a bare word match drive a semantic verdict. Pair the lexicon regex with a **structural
discriminator**: correlate the claimed verb to the **specific tool's** observed success/attempt this turn. A
"could not <verb>" is only false if that verb's tool actually succeeded; a policy refusal (tool never
attempted) stays silent. The τ² personas can be non-English, so word-search alone is unsafe — the
adversarial N1 (magnet) reviewer specifically hunts for reply-regex that proxies intent.

**Long-term:** avoid word-search for semantic detection entirely (structural signals, or generation that
emits language-correct / multilingual patterns).
