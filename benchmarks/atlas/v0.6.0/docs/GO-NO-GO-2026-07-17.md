# GO / NO-GO — does the governed/looprun governance skill generate value vs a plain frontier-LLM Mastra agent?

**Date:** 2026-07-17 · **Status:** definitive report (standalone). Sources cited inline.
**Subject:** `atlas` (Atlas Equipment Rentals & Field Ops — 5 agents / 61 cases / 54 tools).
**Ruler:** the certified atlas 61-case eval set, Claude judge (ruler-v2).
**Terminology:** **governed** = the governed arm produced by the agentspec-generator skill (becomes
"looprun" after the mirror). **vanilla** = the blind Mastra arm, authored by a separate frontier-LLM
session that never saw looprun, GUARDS.md, the specs, the eval cases, the rubrics, or the world model
(provenance: `../vanilla/agents-generated/atlas/PROVENANCE.md`).

---

## 1. Verdict — GO (qualified)

**GO.** The governance skill delivers real, measurable value — but that value is **concentrated, not
uniform**. On a strong cloud model the ungoverned vanilla agent ties or slightly edges the governed
one on aggregate quality; the skill's win shows up decisively on the **weak/local tier (+6.5 pt)**, in
**zero deterministic fails** on money/irreversible/happy-path actions, in **holding every tier from one
spec source** (where vanilla must trade one tier for another), at **−37% / −29% lower cost**, and in a
**repeatable local-tuning process** plus DX≈0 generated world/evals/certification the vanilla path never
produces. Ship it, and pitch it honestly as a weak-tier + safety + cost + single-source win — not a
strong-model quality win.

## 2. The number, per tier

Measurement law (user, 2026-07-17; codified as **D25**, `docs/tables/decisions.md`): guards are
identical across models (deterministic); only the **prose** is optimized per model. So count, per arm
per tier, **only the number from the spec optimized FOR that tier** — cross-tier numbers (a spec
measured on a model it was not tuned for) are discarded.

| tier | governed (skill) | vanilla (blind) | Δ |
|---|---|---|---|
| **FL** (flash-lite, cloud) — each arm's FL-optimized spec | **100% N=3** (61/61 ×3, r2) | 98.4% (v2, N=1) | governed +1.6 |
| **ram24** (local, quantized IQ2_XXS) — each arm's ram24-optimized spec | **93.4% N=3** (r2, MTP-on, byte-deterministic; perturbed band in flight) | 86.9% (v3, N=1, MTP-on) | **governed +6.5** |

Current governed state = the **r2 bundle, ONE spec source**: FL 100% N=3 · ram24 93.4% N=3
(`atlas2-regen-report-2026-07-17.md`, "r2" section + "r2 ram24 N=3 (MTP-on) — closed"). r1 (the
prior from-scratch bundle) was FL 96.7% N=3 / ram24 93.4% N=1+band, MTP-off
(`flip-root-cause-2026-07-16.md` Part 6); r2's revised STOP rule lifted FL to 100% and recovered the
MTP-on penalty without regression.

**Discarded as cross-tier** (spec on the wrong model): vanilla v2-ram24 (85.2%), vanilla v3-FL
(95.1%); vanilla v1 (FL 85.2 / ram24 72.1, superseded). governed r2's prose was iterated against BOTH

(`atlas2-regen-report-2026-07-17.md`, measurement-law table).

**Config note (materiality).** MTP is a ram24-only serving flag; FL (cloud) is MTP-irrelevant. On r1,

**prose-dependent, not intrinsic**. Certify and pin the serve config you deploy.

## 3. Where the value IS and ISN'T

**Honest boundary first — where it ISN'T.** The skill does **not** buy strong-model aggregate quality.
On the FL cloud tier the ungoverned, hand-tuned vanilla agent is within noise of the governed one
(98.4% vs the governed arm's 96.7% at r1; r2 closes and passes it to 100%, but the FL delta is small
either way). This reconfirms the earlier-subject precedent: on a strong model, good iron-rule prose from a
capable frontier LLM matches governance on the raw pass-rate. Do not sell governance as a
strong-model quality lift.

**Where it IS — six concentrated wins:**

1. **Weak / local models: +6.5 pt.** ram24 governed 93.4% vs vanilla 86.9%. Governance pays the most
   exactly where the model is weakest — the deployment tier where a plain agent degrades most.
2. **Zero deterministic fails on money / irreversible / happy-path actions.** The governed arm hit **zero
   autofails** across all runs; the vanilla arm hit **invariant autofails on 01, 08, 22, 27, 64, 88**
   across its runs (`atlas2-regen-report-2026-07-17.md`, VERDICT table). Guards block the
   fabricate/skip-a-gate failures the ungoverned arm commits.
3. **One spec source covers multiple tiers.** governed gets FL 100% AND ram24 93.4% from a **single r2
   bundle**. Vanilla cannot hold both from one bundle: **v2 is FL-strong / local-weak (98.4 / 85.2),
   v3 is the reverse (95.1 / 86.9)** — a per-tier tradeoff that cancels. The blind dev pays a full
   iteration per tier; the skill holds both from one source (+ cheap per-tier profiles if wanted).
4. **Cost: −37% FL / −29% ram24 tokens per case** (measured; `atlas2-regen-report-2026-07-17.md`).
5. **The near-tie / margin technique makes local improvement a PROCESS, not luck** (see §4). Vanilla's
   local tuning is a coin-flip (fix one case, break another — structural, `flip-root-cause` Part 1);
   the skill converts it to a deterministic, ~1-minute, zero-judge-call decision.
6. **DX≈0 + generated world / evals / certification.** The skill produces the FakeWorld, the eval
   ruler, and a banded local certification end-to-end — artifacts the vanilla path never generates.
   And the vanilla arm was still handed `tools.json` + a sanitized brief **for free** (a real cost in
   practice; PROVENANCE.md), so this comparison is already generous to vanilla.

## 4. The technique — near-tie margin discipline + revised STOP rule

This is what let the skill reach FL 100% and recover the MTP-on local number **without whack-a-mole**
(codified as **D25**; measured in `flip-root-cause-2026-07-16.md`).

- **Root cause of local flips:** a graded action rides one greedy-argmax token whose margin (~0.3–0.5
  nat) is the residual of a prose equilibrium, under two noise sources each ≥ the margin (prose ±0.35
  nat/byte, cache-state ±0.21 nat; Parts 1–2). So an N=1 score A/B measures `edit + noise` where noise
  is the same order as the signal — "enxugar gelo" (bailing water) is **structural, not bad luck**.
- **The instrument:** a fork-pair (`extract-fork.mjs` + `margin-probe.py`) reads top-k logprobs at the
  decision fork on the target engine (speculative OFF) and accepts an edit only if the **worst-case
  margin across a noise battery improves on both the primary and mirrored-intent fork** (target ≥3×
  the band). It caught 2 score-passing edits as regressions, deterministically, in ~1 min each, zero
  judge calls (Part 2).
- **The revised STOP rule (bar = floor):** stop AT the bar → keep iterating while each round nets
  positive (to the 3-iteration cap), but past the floor the **only** admissible fixes are
  margin-validated prose or deterministic gates — never blind prose. r2 = r1 + targeted emphatic
  iron-rule prose on the residual language-layer cases → FL 100% N=3 and ram24 93.4% MTP-on, **net
  positive on both tiers, zero sibling regression** (`atlas2-regen-report-2026-07-17.md`, r2 section).

## 5. Caveats / open

- **Frontier tier unmeasured.** The OpenRouter frontier matrix has not been run for the vanilla arm —
  the strong-model story could shift there (in either direction).
- **The local certification band is still closing.** ram24 N=3 is byte-deterministic (reps identical),
  so the honest error bar is the **perturbation band** (r1 measured 1-flip under an inert byte); the
  perturbed band for r2 is in flight.
- **Vanilla got its inputs for free.** `tools.json` + a sanitized brief were handed to the blind
  session (PROVENANCE.md) — a real cost in practice, so the comparison already favors vanilla.
- **Single subject.** All numbers are atlas only. Criaty stays the regression anchor; never compare
  pass-rates across subjects.
- **Claude baseline is a floor, not a comparison.** The A3 Claude-as-agent run = 61/61, but r1/r2
  prose was not Claude-optimized, so by the measurement law it is a **labeled day-0 floor**, not a
  per-tier cert number.
- **Robustness cross-check (hardening, not a caveat):** the governed bundle scored **93.4% first-shot,
  un-iterated on a FRESH independent ruler (atlas2)** it never saw — same band as its first-shot on
  the certified ruler. This hardens the result against "the ruler was tuned to favor governed"
  (`atlas2-regen-report-2026-07-17.md`, Item 4).

## 7. ADDENDUM (2026-07-18): the cloud/frontier tier — verdict CONFIRMED and sharpened

The declared pendency (frontier tier via the OR matrix) is closed:
**13 cloud models, full-61, N=3 both arms, same post-fix evals, same Opus ruler** —
full data + forensics in [`vanilla-cloud-matrix-2026-07-18.md`](vanilla-cloud-matrix-2026-07-18.md).

- **Aggregate: governance +3.9 pt** (governed 96.5 vs vanilla 92.6); **11 wins / 1 tie (luna) /
  1 loss (nano −6.0)**. The v1-era "governance ≈ 0 on strong models" now reads: only the very top
  (haiku/sonnet/ds-flash) come within ~2 pt.
- **Every risk-class incident in the matrix — fabrication (8 models), one-shot destructive,
  delivery-stub (grok +13.1), privilege-escalation ask, tenant-isolation offer, PII dump —
  happened in the UNGOVERNED arm.** Screen ceilings glm-4.7-flash/mimo (66.7%) fail on exactly
  these; mimo's governed profile scored 97.3 = ~30 pt of pure governance.
- **Rep-stability is part of the product:** ungoverned reps swing up to ±6.6 pt same-config;
  governed reps were stable (locally ×3 byte-identical, D20).
- **The honest exception is a lesson, not a hole:** nano's inversion is the confirm-gate acting as
  an ATTRACTOR for a literal model on sibling-scope requests — fixed by scope-precedence prose in
  the skill (+ the recoverable reply-delivery stub fix), NOT by less governance. terra (+4.9)
  refutes any vendor-wide "OpenAI inversion".
- **Cost:** the governed arm stays the cheaper per case; the whole vanilla cloud matrix cost
  US$ 57.87 (upper-bound pricing).

**Pitch line, updated:** on the frontier tier the hand-tuned agent no longer matches — governance
wins the aggregate by ~4 pt, wins 11 of 13 models, and is the only arm with zero fabrication,
zero unconfirmed destructive executions, and stable replicates — while remaining cheaper.
